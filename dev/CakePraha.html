Cake Praha

Make your application i18n and speak on your users languages

>> haxelib install CakePraha

See all libs and tutorials

Prerequirements

You need to put text file with ".txt" or ".csv" extension to "i18n" or any other localization dir in assets directory. You can also add "index" file with list of available locales.

We use CSV (comma-separated values) format for all localization strings. Each string should be in new line. You can use "%DIGIT%" format format for specifying dynamic values.

There is two special keys can be in each locale:

  • "%RightToLeft" - set it to TRUE if text should be displayer in right-to-left format
  • "%font" - specify it if you need to use special font for current locale

Here is a sample of typical index file ("assets/i18n/index.csv"):
"en","ru","ua", "non-exist"
Here is a sample of typical locale file ("assets/i18n/en.csv"):
"%RightToLeft", "No"
"%font", "Arial"
"test","Hello %1%!\n%2%"
"Good" , "Good day, sir %1%!"

Check "demo" folder for working sample

locale:String

All platforms

Store current locale name. Correct locale automaticly load if you change this value

isRightToLeft:Bool

All platforms

Flag represent that current locale is right-to-left display format

localeFont:String

All platforms

Store font name recomended for this locale, if specified

paths:Array<String>

All platforms

Array of default localization files search pathes. You can change it if you use some specific path for your localization files.

By defult, it's equal to ["i18n","i18","localization","locales","lang","langs"]

getLocales ( )

return:Array<String>

All platforms

Return list of known and available locales. By default, values are taken from index file if it's exists.

getLine ( )

return:String

All platforms

Return localized string

name:String – string name

?keys:Array<String> – array of strings, that should be putted instead of %1%,%2%,%3%... marks
